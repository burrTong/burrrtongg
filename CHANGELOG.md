# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

### Added
- Complete database initialization with `init.sql` containing baseline data
- 7 test coupons with various scenarios (active/inactive, expired, limited use, minimum purchase)
- 3 sample orders with different statuses (PENDING, DELIVERED, CANCELED)
- Flyway + init.sql hybrid approach for database management
- Development environment setup script (`scripts/init_dev_environment.ps1`)
- Comprehensive documentation (`DEV_SETUP.md`, `BASELINE_DATA.md`)
- Bind mount for product images (persists across container rebuilds)

### Changed
- Switched from Flyway-only to Flyway + init.sql hybrid approach
- Updated `application.properties` to disable Flyway checksum validation
- Changed coupons table from `SERIAL` to `BIGINT GENERATED BY DEFAULT AS IDENTITY`
- All tables now use `BIGINT` for ID columns consistently
- Product images now use bind mount instead of named volume

### Fixed
- UTF-8 encoding issues in `init.sql` (removed BOM)
- Schema validation errors for coupons table data type mismatch
- SEQUENCE synchronization with actual data

### Technical Details
- Database: PostgreSQL 15.14
- Backend: Spring Boot 3.5.5, Java 21, Flyway (validate-on-migrate=false)
- Frontend: React + Vite on port 5173
- Elasticsearch: 8.11.1 for product search

## [0.1.0] - Initial Setup

### Added

- User authentication (Customer, Admin)
